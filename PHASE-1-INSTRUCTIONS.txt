WHAT THIS PHASE DOES
1. Put the starter app files into GitHub.
2. Grab the Supabase anon key.
3. Add the 2 environment variables into Vercel.
4. Deploy the shell.
5. Confirm the site opens and the 3 placeholder pages work.

STEP 1 - UPLOAD THE STARTER FILES TO GITHUB
Exact repo:
https://github.com/TraderMigs/NotaryTool/upload/main

1. Open the zip file from this package on your computer.
2. Open the folder named notarytool-phase1.
3. Select everything inside that folder.
4. Drag all of it into your GitHub upload page.
5. In the commit title box, type:
   Phase 1 starter shell
6. Click the green Commit changes button.

STEP 2 - GET YOUR SUPABASE ANON KEY
Exact project settings area:
https://supabase.com/dashboard/project/ikhvtktvsvtfmzyrypto/settings/api

1. Open the Supabase API settings page above.
2. Find Project URL.
3. Find anon public key.
4. Keep that browser tab open.

STEP 3 - ADD ENVIRONMENT VARIABLES IN VERCEL
Exact import page shown in your screenshot:
https://vercel.com/new/import

1. Open your NotaryTool project import page in Vercel.
2. Make sure the project name says notary-tool.
3. Under Application Preset, choose Next.js if it appears. If Next.js is not available yet, Other is still okay because the files include package.json and Next config.
4. Open the Environment Variables section.
5. Add this first variable:
   Name: NEXT_PUBLIC_SUPABASE_URL
   Value: https://ikhvtktvsvtfmzyrypto.supabase.co
6. Add this second variable:
   Name: NEXT_PUBLIC_SUPABASE_ANON_KEY
   Value: paste your anon public key from Supabase.
7. Leave everything else alone.
8. Click Deploy.

STEP 4 - WAIT FOR THE FIRST DEPLOY TO FINISH
1. Let Vercel build.
2. When it finishes, open the deployed site.
3. You should see the NotaryTool home page.
4. Click Sanitize.
5. Click Review.
6. Click Dashboard.

STEP 5 - WHAT SUCCESS LOOKS LIKE
You are done with Phase 1 when all 4 things are true:
1. GitHub contains the uploaded files.
2. Vercel deployment is successful.
3. The home page loads.
4. The 3 internal pages load without errors.

IMPORTANT
This phase is only the shell.
There is no real upload logic yet.
There is no real PDF processing yet.
That starts in Phase 2.
